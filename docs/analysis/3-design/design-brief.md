## Цель и контекст

**«Мимо Кассы»** заменяет ручную запись на кулинарные классы через WhatsApp и Google-таблицу,
устраняя двойные брони и путаницу с местами. Клиент самостоятельно просматривает слоты,
фильтрует, записывается (себя и гостей), выбирает вариант экипировки, отменяет записи и получает
напоминания. Платформа — **нативное мобильное приложение для iOS** (mobile-first для использования
с телефона), распространяется через App Store (NFR-4). Веб-версии в скоупе нет.

**Скоуп — только роль «Клиент».** Шеф и владелец работают через существующую
инфраструктуру/админку и в приложение не входят. Справочные данные — read-only из API;
оплата — офлайн (наличные / перевод на карту), приложение лишь показывает цену и фиксирует запись.

**Что входит в MVP (принятые решения):**
- **Профиль клиента (R-009):** просмотр и редактирование имени и телефона, выход из аккаунта,
  удаление аккаунта — экран [SCR-007](SCR-007-profile.md).
- **Цена и прокат (R-010):** прокат экипировки — **отдельный тариф**; итог записи =
  `цена за место × число мест + тариф проката × число прокатных комплектов`. **Своя экипировка —
  бесплатно.** Валюта — **рубли (RUB)**. Итоговую цену считает и возвращает сервер; приложение
  показывает её, не пересчитывая.
- **Адрес студии (R-011):** обязательный адрес студии + карта с меткой на
  [SCR-003](SCR-003-slot-card.md)/[SCR-006](SCR-006-booking-details.md); при недоступности
  карты — текстовый fallback (см. [00-foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты)).
- **Отмена (R-014):** только **целиком** (нельзя отменить часть мест из записи).
- **Оценки шефов — Phase 2 (R-001):** осознанно вынесены из MVP и перенесены в Phase 2
  (согласовано с заказчиком).

Подробнее о продукте, аудитории, дизайн-принципах и контексте использования —
[00-foundations.md §1–2](00-foundations.md#1-продукт-и-аудитория).

## Роли и их задачи

| Роль | Что делает в приложении |
| :-- | :-- |
| **Клиент** | Просматривает и фильтрует слоты, записывается (себя + гости), выбирает вариант экипировки, смотрит цену, отменяет записи, видит свои брони, получает напоминания. |
| Шеф / Владелец | **Не входят в приложение** — работают через существующую инфраструктуру/админку (вне скоупа). |

## Сквозные правила (foundations)

Единые для всех экранов требования вынесены в **[00-foundations.md](00-foundations.md)**:
дизайн-принципы, структурные токены (контраст, тач-зоны, иерархия — без бренда), каркас и
навигация (таб-бар, bottom sheet, нижний CTA), сквозной паттерн состояний
(Loading / Content / Empty / Error), tone of voice и общая микрокопия, доступность,
напоминания, безопасность данных, глоссарий.

> **Объём визуальных требований.** Документы задают **функционально-структурные** требования
> (иерархия, компоненты, поведение, ограничения), без прескриптивных цветов/шрифтов — бренд
> в исходной аналитике не зафиксирован; визуальный язык определяет дизайнер.

## Пользовательские сценарии (user flows)

Полная карта навигации — в [фича-листе §3](../5-mobile-app-spec/feature-list.md). Ключевые пути:

- **Вход (трёхшаговый):** запуск → SCR-001 (телефон → код из SMS → имя для нового) → SCR-002.
- **Запись (UC-1):** SCR-002 → SCR-003 → SCR-004 → BS-002 → (SCR-005). ≤ 3 экранов до подтверждения (NFR-2). После **первой** записи в BS-002 запрашивается разрешение на push.
- **Фильтрация (UC-3):** SCR-002 → BS-001 → SCR-002 (с применёнными фильтрами; множественный выбор типа программы / шефа).
- **Отмена (UC-2):** SCR-005 → SCR-006 → BS-003 → SCR-006 (с обновлённым статусом). Отмена — только целиком.
- **Профиль:** SCR-007 — просмотр/редактирование имени и телефона, выход, удаление аккаунта, справочные пункты; выход/удаление → SCR-001.
- **Карта студии:** SCR-003 / SCR-006 → BS-004 (интерактивная карта с меткой студии) → возврат на исходный экран.

> **Карта студии и адрес.** На [SCR-003](SCR-003-slot-card.md) и [SCR-006](SCR-006-booking-details.md)
> адрес студии показывается **картой Яндекс (статичный превью с меткой студии)**, а
> **адрес студии** — обязательный элемент (метка на карте + текстовый адрес). Тап по карте
> открывает шторку [BS-004 «Карта студии»](BS-004-studio-map.md) с открытой интерактивной картой
> и меткой студии. Правила — [00-foundations.md §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты).

## Тон, бренд, ограничения

- **Тон:** простой, прямой, дружелюбный, без давления и штрафов; обращение на «вы» —
  [00-foundations.md §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).
- **Бренд:** не зафиксирован в аналитике — визуальный язык за дизайнером; обязательны лишь
  структурные ограничения (контраст, размеры, иерархия) — [§3](00-foundations.md#3-структурные-токены-без-бренда).
- **Доступность / mobile-first:** крупные тач-зоны, высокий контраст, читаемость — [§3, §7](00-foundations.md#7-доступность-nfr-1).
- **Что НЕ делаем на старте (Phase 2+):** онлайн-оплата, оценки шефов (согласовано с заказчиком),
  программа лояльности, публичные отзывы/рейтинги, интерфейсы шефа/владельца, SMS/email-напоминания
  (только push), «поделиться» классом — см. [фича-лист §7](../5-mobile-app-spec/feature-list.md).

## Ключевые ограничения MVP (напоминание дизайнеру)

| Ограничение | Значение |
|-------------|----------|
| **Студия** | 12 рабочих столов |
| **Потолок группы** | Простые классы — до 12 человек; сложные — до 8 человек |
| **Прокатная экипировка** | Всего 12 комплектов (фартук + ножи) |
| **Правило отмены** | Ранняя отмена — за 24 часа и более; поздняя — менее 24 часов, штрафов нет |
| **Максимум мест в одной брони** | 3 (себя + до 2 гостей) |
| **Оплата** | Только офлайн (наличные / перевод на карту) |
| **Канал напоминаний** | Только push (SMS/email — Phase 2) |
