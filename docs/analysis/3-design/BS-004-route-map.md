## 1. Назначение и контекст

Шторка даёт клиенту **рассмотреть расположение студии на полноразмерной интерактивной карте** и
понять, **куда приезжать** для участия в классе. Это раскрытие статичного превью карты с
[SCR-003](SCR-003-slot-card.md) / [SCR-006](SCR-006-booking-details.md): на превью адрес студии виден
обзорно, в BS-004 его можно **приблизить, подвигать** и при необходимости **открыть в приложении
Яндекс.Карты** или **проложить маршрут до студии**.

Контекст использования — клиент может записываться на ходу, часто с телефона (см. [foundations §1](00-foundations.md#1-продукт-и-аудитория)):
карта крупная, элементы управления — под палец, ключевая текстовая информация (адрес студии)
продублирована текстом и читается на ходу.

BS-004 — **вспомогательный просмотр**, не критичное подтверждение: закрытие свободным жестом
(свайп / тап по бэкдропу) допустимо.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** «Хочу понять, где находится студия и как до неё добраться» — рассмотреть
  расположение студии на карте, найти адрес, при желании построить маршрут до неё.
- **Цель продукта:** снять неопределённость «куда ехать» до класса, не уводя клиента из приложения
  без необходимости ([US-4, US-17](../2-requirements/user-stories.md)).

---

## 3. Навигация

**Входящие переходы:**
- Тап по **блоку карты студии** (или действию «Открыть карту») на
  [SCR-003 · Карточка класса](SCR-003-slot-card.md).
- Тап по **блоку карты студии** на [SCR-006 · Детали брони](SCR-006-booking-details.md).

**Исходящие переходы:**
- **Закрытие** (кнопка «Закрыть» / свайп вниз / тап по бэкдропу) → возврат на исходный экран
  ([SCR-003](SCR-003-slot-card.md) или [SCR-006](SCR-006-booking-details.md)) **без изменений**.
- **«Открыть в Яндекс.Картах»** → передача адреса/координат во внешнее приложение или веб
  Яндекс.Карт (handoff за пределы приложения).
- **«Проложить маршрут»** → внешняя навигация Яндекс.Карт к адресу студии (turn-by-turn — вне
  скоупа приложения, выполняется внешним приложением).

**Свойства шторки:**
Общие правила bottom sheet (бэкдроп, грабер, swipe-to-close, анимация, скрытие таб-бара) —
[foundations §4.3](00-foundations.md#43-bottom-sheet-шторки-bs-001--bs-002--bs-003). Специфика
BS-004: высота — **расширенная** (до ~90% экрана), чтобы карта была крупной; контент карты
интерактивен (зум/пан) внутри шторки.

## 4. Контент-инвентарь

Значения приходят из данных слота/студии (read-only). Числа и координаты **не хардкодятся**.

| # | Элемент | Источник данных | Назначение |
|---|---|---|---|
| 1 | Интерактивная карта | — (Яндекс Maps JS API) | Полотно карты с зумом/паном. |
| 2 | Метка студии | `slot.studio_lat` / `slot.studio_lng` | Метка на карте с адресом студии. |
| 3 | Название студии | `slot.studio_name` (или константа «Шеф-стол») | Название студии (текстом). |
| 4 | Адрес студии | `slot.studio_address` | Текстовый блок «куда ехать». |
| 5 | Кнопка «Открыть в Яндекс.Картах» | — | Handoff во внешнее приложение/веб Яндекс.Карт. |
| 6 | Кнопка «Проложить маршрут» | — | Навигация до студии во внешнем приложении. |
| 7 | Кнопка «Закрыть» / грабер | — | Закрыть шторку, вернуться на исходный экран. |

> Компонент карты (метка + источник тайлов/ключ Яндекс) описан в
> [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты); BS-004 —
> его **интерактивная** версия.

## 6. Компоненты и поведение

### 6.1 Интерактивная карта
- Полотно Яндекс Maps JS API: **зум и пан** жестами; стартовый вид подобран так, чтобы метка
  студии **помещалась в центре кадра**.
- **Метка студии** — заметная метка с адресом/названием; по тапу — короткая подпись (адрес студии).
- Контролы зума (＋/−) — крупные тач-зоны ([foundations §3.1](00-foundations.md#31-тач-зоны-и-размеры)).
- Ключ API и стиль тайлов — параметр конфигурации (foundations §4.5), в макет не зашиваются.

### 6.2 Текстовый блок (эквивалент карты)
- Название студии и **адрес студии** — текстом.
- Служит **текстовым эквивалентом** карты для доступности (§9) и для случая, когда карта недоступна (§7).

### 6.3 Кнопки действий
- **«Проложить маршрут»** — открывает внешнюю навигацию Яндекс.Карт к адресу студии
  (`studio_lat/lng`). Turn-by-turn выполняется внешним приложением (вне скоупа).
- **«Открыть в Яндекс.Картах»** — handoff: открывает адрес студии во внешнем приложении или вебе.
- Обе — крупные тач-зоны; имеют доступные имена.

### 6.4 Поведение bottom sheet
- Расширенная высота (до ~90%), грабер, бэкдроп, swipe-to-close — по
  [foundations §4.3](00-foundations.md#43-bottom-sheet-шторки-bs-001--bs-002--bs-003).
- Закрытие тапом по бэкдропу / свайпом / кнопкой «Закрыть» ведёт к одному исходу — возврату на
  исходный экран. BS-004 — **не критичное подтверждение**, свободный жест закрытия допустим.

## 7. Состояния экрана

Сквозной паттерн — [foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана) и
правила карты [§4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты). Специфика BS-004:

| Состояние | Содержимое / Поведение |
|---|---|
| **Loading (инициализация карты)** | Скелетон/индикатор на месте карты; текстовый блок (название и адрес студии) уже виден. |
| **Content** | Интерактивная карта с меткой студии + текстовый блок + действия. Основной режим. |
| **Error / offline (карта не загрузилась)** | Вместо карты — нейтральное сообщение + кнопка **«Обновить»**; **текстовый блок адреса студии** и кнопка **«Открыть в Яндекс.Картах»** остаются доступны. Шторка не ломается. |

## 8. Валидации и микрокопия

BS-004 не содержит вводимых полей. Контентные правила:

| Контекст | Текст |
|---|---|
| Заголовок | «Адрес студии» |
| Лейбл адреса | «Адрес студии» |
| Кнопка навигации | «Проложить маршрут» |
| Кнопка внешней карты | «Открыть в Яндекс.Картах» |
| Кнопка закрытия | «Закрыть» |
| Ошибка загрузки карты | «Не удалось загрузить карту. Проверьте соединение и попробуйте снова.» ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)) |

- Тон — простой, без давления ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).
- Адрес студии и название — из данных, без хардкода.

## 9. Доступность и mobile-first

Базовые правила — [foundations §7](00-foundations.md#7-доступность-nfr-1),
[§3.1](00-foundations.md#31-тач-зоны-и-размеры), [§4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты). Специфика BS-004:

- **Карта — не единственный носитель информации:** адрес студии продублирован **текстом**;
  карта и метка имеют доступные имена.
- При недоступности карты текстовый блок и действие «Открыть в Яндекс.Картах» остаются (§7).
- **Крупные контролы** (зум, кнопки действий, «Закрыть») с достаточными отступами.
- Контраст текста — не ниже WCAG AA.
- Layout не ломается при системном увеличении шрифта; карта скроллится/масштабируется внутри шторки.

## 10. Критерии приёмки дизайна

```gherkin
Функция: Шторка карты студии (BS-004)

  Сценарий: AC-001 — Открытие карты с карточки класса
    Дано клиент на SCR-003 и видит блок карты студии
    Когда он тапает по карте
    Тогда снизу открывается шторка BS-004 с интерактивной картой
    И на карте отрисована метка студии

  Сценарий: AC-002 — Открытие карты из деталей брони
    Дано клиент на SCR-006 и видит блок карты студии
    Когда он тапает по карте
    Тогда открывается шторка BS-004 с той же меткой

  Сценарий: AC-003 — Зум и пан
    Дано открыта шторка BS-004
    Когда клиент масштабирует и двигает карту
    Тогда карта реагирует на жесты, метка студии остаётся отрисованной

  Сценарий: AC-004 — Текстовый блок адреса
    Дано открыта шторка BS-004
    Тогда показаны название и адрес студии текстом

  Сценарий: AC-005 — Открыть в Яндекс.Картах / проложить маршрут
    Дано открыта шторка BS-004
    Когда клиент нажимает «Открыть в Яндекс.Картах» или «Проложить маршрут»
    Тогда происходит переход во внешнее приложение/веб Яндекс.Карт с этим адресом

  Сценарий: AC-006 — Закрытие возвращает на исходный экран
    Дано шторка BS-004 открыта с SCR-003 (или SCR-006)
    Когда клиент закрывает её (кнопка/свайп/бэкдроп)
    Тогда он возвращается на исходный экран без изменений

  Сценарий: AC-007 — Карта недоступна (offline/ошибка)
    Дано карта не загрузилась
    Тогда показано нейтральное сообщение с «Обновить»
    И остаются доступны текстовый блок адреса и «Открыть в Яндекс.Картах»
```

## 11. Решения

| # | Решение |
|---|---|
| 1 | **BS-004 — отдельная шторка** с интерактивной картой; открывается по тапу на превью карты ([foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты)) с SCR-003/SCR-006. |
| 2 | **Внешняя навигация — handoff в Яндекс.Карты** («Открыть в Яндекс.Картах», «Проложить маршрут»); turn-by-turn внутри приложения — вне скоупа MVP. |
| 3 | **Высота шторки — расширенная** (до ~90%), чтобы карта была крупной; не критичное подтверждение → свободное закрытие. |
| 4 | *(Параметр конфигурации, не блокер)* **Ключ Яндекс Maps JS API / Static API** и стиль тайлов — задаются конфигурацией. |
