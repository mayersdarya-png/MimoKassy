## 1. Назначение и контекст

Экран показывает **полную информацию об одной записи (брони)** клиента и служит точкой
**запуска её отмены**. Это вложенный экран, открываемый из списка [«Мои бронирования»
(SCR-005)](SCR-005-my-bookings.md) тапом по конкретной записи.

Две задачи экрана:
1. **Просмотр.** Собрать в одном месте все параметры брони и её текущий статус, чтобы клиент
   понимал, на что и когда он записан, сколько мест и какую экипировку выбрал, какую цену оплатить офлайн.
2. **Отмена.** Дать единственное действие над записью — отменить участие. Само подтверждение
   и пояснение последствий (правило 24 часов) выполняется в шторке [BS-003](BS-003-cancel-confirm.md),
   экран лишь запускает её и отображает результат после возврата.

Контекст использования — запись на ходу, часто с телефона, одной рукой
(см. [foundations §1](00-foundations.md#1-продукт-и-аудитория)):
статус и ключевые числа читаемы с первого взгляда, деструктивное действие защищено подтверждением.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** убедиться в деталях своей записи и при необходимости отменить её
  заранее, понимая последствия.
- **User story:** [US-10](../2-requirements/user-stories.md) — как клиент, я хочу видеть детали
  своей брони и отменять её, чтобы освободить место, если мои планы изменились.
- **Сценарий:** [UC-2 «Отмена записи»](../2-requirements/use-cases.md) (основной поток, A1
  поздняя отмена, исключения E1/E2).
- **Принцип «только свои данные»** ([foundations P5](00-foundations.md#2-дизайн-принципы)):
  экран показывает данные исключительно текущего клиента.

## 3. Навигация

**Входящие переходы:**
- Тап по карточке записи в списке [«Мои бронирования» (SCR-005)](SCR-005-my-bookings.md).

**Исходящие переходы:**
- Кнопка **«Отменить»** (нижний CTA) → открывает шторку [BS-003 «Подтверждение отмены»](BS-003-cancel-confirm.md).
  После подтверждения отмены управление возвращается на SCR-006 с обновлённым статусом.
- **Тап по карте студии / «Открыть карту»** → шторка [BS-004 «Карта студии»](BS-004-studio-map.md)
  (открытая интерактивная карта с меткой студии).
- **Назад** (в хедере) → возврат на [SCR-005](SCR-005-my-bookings.md).

Таб-бар на этом экране **скрыт** — экран вложенный (см. [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона)).
Полная карта переходов — в [фича-листе §3](../5-mobile-app-spec/feature-list.md).

## 4. Контент-инвентарь

Источник полей — [модель данных](../4-design/data-model.md) (сущность `booking` и связанные
`slot`, `program`, `chef`). Конкретные имена полей сверяются с моделью при детализации.

| Блок | Поле / данные | Откуда | Примечание |
|------|---------------|--------|------------|
| Статус | `status` | booking | Активна / отменена / поздняя отмена / отменена студией / прошедшая (бейдж). |
| Когда | `slot.start_at` | slot | Дата и время старта класса. |
| Программа | `slot.program.name` + `program.type` | program | Название программы и её тип (простая / сложная). |
| Карта студии | `slot.studio_address` + координаты | slot | **Статичный превью Яндекс-карты с меткой студии** (компонент [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты)). Тап → шторка [BS-004](BS-004-studio-map.md) (интерактивная карта). |
| Адрес студии | `slot.studio_address` | slot | **Обязательный.** Метка на карте + текстовый блок (адрес) под картой. |
| Шеф | `slot.chef.name` | chef | Кто ведёт класс. |
| Цена | `slot.price_total` или `slot.price` | booking / slot | Стоимость; оплата офлайн (текст — из foundations §6). |
| Места | `seats_count` | booking | Сколько мест в брони (себя + гости). |
| Экипировка | вариант экипировки по местам / `rental_count` | booking | Своя / прокатная; число прокатных комплектов. Лейблы — из foundations §6. |
| Создана | `created_at` | booking | Когда сделана запись. |
| Отменена | `cancelled_at` | booking | Показывается **только** если запись отменена. |
| Причина отмены студией | `cancellation_reason` | booking | Показывается **только** если статус `cancelled_by_studio`. |

> Числовые лимиты (потолок программы, размер фонда) в тексты **не зашиваются** — берутся из
> данных слота/программы ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).
> Лейблы «Своя экипировка» / «Прокатная экипировка» — из той же таблицы микрокопии.

## 5. Структура и иерархия

Каркас следует [foundations §4.1](00-foundations.md#41-базовый-каркас): фиксированный хедер,
скролл-контент, фиксированный нижний CTA. Таб-бар отсутствует.

- **Хедер (фикс.):** кнопка «Назад» + заголовок экрана («Детали записи»).
- **Скролл-контент:** бейдж статуса (вверху, заметный) → блок «Когда» (дата/время старта) →
  блок «Программа и шеф» → **карта студии + адрес** → блок «Места и экипировка» →
  блок «Цена» (+ напоминание об офлайн-оплате) → служебные метки (создана / отменена / причина).
- **Фикс. нижний CTA:** кнопка **«Отменить»** во всю ширину контентной области. При недоступности
  действия CTA **показывается неактивным (disabled) с пояснением** причины (см. §6, §7).

## 6. Компоненты и поведение

### 6.1 Блоки деталей
Сгруппированные секции/карточки (одна колонка, [foundations §3.4](00-foundations.md#34-плотность-и-сетка)).
Каждая секция имеет заголовок-лейбл и значение; ключевые числа (старт, цена, мест) — крупные и
контрастные ([foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)). Read-only,
без редактирования.

### 6.2 Бейдж статуса
Отражает `status` записи. Состояния не передаются **только цветом** — обязательны иконка/текст
([foundations §3.2, §3.5](00-foundations.md#35-иконки-и-индикаторы)). Возможные значения:
- **Активна** — предстоящая, не отменённая запись.
- **Отменена** — ранняя отмена (места освобождены).
- **Поздняя отмена** — отменена < 24 ч до старта (место не освобождено). Текст-итог — из
  [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).
- **Отменена студией** — класс отменён организатором (срыв поставки, поломка оборудования).
- **Прошедшая** — старт уже наступил.

### 6.3 Кнопка «Отменить» (нижний CTA)
- Крупная, во всю ширину ([foundations §3.1](00-foundations.md#31-тач-зоны-и-размеры)).
- **Enabled** — только когда запись **активна** И класс **ещё не начался**. Тап → открывает
  [BS-003](BS-003-cancel-confirm.md).
- **Disabled (показывается неактивной, не скрывается)** — если класс уже стартовал/прошёл
  (**UC-2 E1**) либо запись уже отменена (**UC-2 E2** — повторная отмена не выполняется). Рядом —
  **обязательное короткое пояснение** причины недоступности (см. §8), чтобы причина была читаема.
- **Отмена — только целиком.** Частичная отмена отдельных мест в MVP не поддерживается (см. §11).

> Деструктивное действие не выполняется на этом экране напрямую — фактическое подтверждение,
> логика ранней/поздней отмены и текст правила 24 часов — в [BS-003](BS-003-cancel-confirm.md).

### 6.4 Карта студии и адрес
- **Карта студии:** статичный превью Яндекс-карты с меткой студии — компонент
  [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты).
  Тап / «Открыть карту» → шторка [BS-004 «Карта студии»](BS-004-studio-map.md) с интерактивной картой.
- **Адрес студии (обязательно):** текстовый блок (адрес студии, `slot.studio_address`) под картой;
  служит текстовым эквивалентом карты (доступность, §9). При недоступности карты — fallback на
  текст + ссылку «Открыть в Яндекс.Картах» (состояния — по §4.5).

## 7. Состояния экрана

Базовый паттерн — [foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).
Специфика SCR-006:

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетон в форме блоков деталей и бейджа статуса (не пустой экран). |
| **Content** | Полные детали брони + статус + нижний CTA по правилам §6.3. |
| **Error** | Заглушка ошибки + кнопка **«Обновить»** (текст — [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)). |
| **Запись отменена** | Бейдж «Отменена» / «Поздняя отмена» / «Отменена студией», показан `cancelled_at` и причина (для студийной отмены). CTA «Отменить» **неактивна с пояснением** «Запись уже отменена» (UC-2 E2). |
| **Класс начался/прошёл** | Бейдж «Прошедшая» (или «Активна» с пометкой о начале). CTA «Отменить» **неактивна с пояснением** «Класс уже начался — отменить запись нельзя.» (UC-2 E1). |
| **Карта: загрузка / ошибка / офлайн** | По [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты): skeleton при загрузке; при ошибке/офлайн — fallback на текстовый блок адреса студии + ссылка «Открыть в Яндекс.Картах». Остальные детали брони остаются доступны. |

Empty state как таковой неприменим — экран всегда про одну существующую бронь; отсутствие
записи трактуется как ошибка/невалидный переход (показ Error).

## 8. Валидации и микрокопия

- **«Отменить» — только до старта (UC-2 E1).** CTA доступен, пока `slot.start_at` в будущем.
  После старта — недоступен. Пояснение в disabled-состоянии: «Класс уже начался — отменить
  запись нельзя.»
- **Повторная отмена не выполняется (UC-2 E2).** Для записи со статусом «отменена» / «поздняя
  отмена» / «отменена студией» CTA скрыт/disabled; показывается текущий статус. Пояснение:
  «Запись уже отменена.»
- **Отмена студией.** Для статуса `cancelled_by_studio` показывается причина отмены
  (`cancellation_reason`), например: «Класс отменён из-за срыва поставки продуктов.»
- **Правило 24 часов — ссылкой, без дублирования.** Рядом с активной кнопкой «Отменить» —
  короткая подсказка на правило отмены; **полный текст и логика ранней/поздней отмены — в
  [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)** (единый источник) и
  раскрываются в [BS-003](BS-003-cancel-confirm.md). Текст правила здесь **не переписывается**.
- **Явное время отсечки.** Рядом с правилом показываем **конкретный момент отсечки** —
  «Бесплатно освободить место можно до `<время старта − 24 ч>`» (значение вычисляется из
  `slot.start_at`, без хардкода). Помогает клиенту сориентироваться, ранняя отмена или поздняя.

Тон — спокойный, без давления и вины ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия), P6).

## 9. Доступность и mobile-first

Базовые требования — [foundations §7](00-foundations.md#7-доступность-nfr-1) и
[§3](00-foundations.md#3-структурные-токены-без-бренда). Специфика экрана:

- **Читаемый статус** — бейдж дублирует смысл текстом и иконкой, не только цветом.
- **Крупный CTA** — кнопка «Отменить» во всю ширину, тач-зона ≥ минимальной; disabled-состояние
  визуально и программно (доступное имя) считывается screen reader.
- **Осторожность с деструктивным действием** — отмена не происходит по одному тапу; обязательное
  подтверждение в [BS-003](BS-003-cancel-confirm.md). Кнопка не должна провоцировать случайное
  нажатие.
- **Карта студии не единственный носитель:** адрес студии продублирован текстом; карта имеет
  доступное имя; при недоступности карты текстовый блок адреса остаётся.
- Layout не ломается при системном увеличении шрифта; ключевые числа остаются крупными и контрастными.

## 10. Критерии приёмки дизайна

```gherkin
Сценарий: AC-001 Показ всех деталей и статуса
  Дано клиент открыл детали своей записи
  Когда экран в состоянии Content
  Тогда отображаются статус, дата/время старта, программа и её тип,
        шеф, число мест, вариант экипировки, цена и дата создания
  И показаны карта студии и адрес
  И для отменённой записи дополнительно отображается дата отмены
  И для отмены студией показана причина отмены

Сценарий: AC-007 Карта студии и адрес
  Дано клиент открыл детали своей записи
  Тогда показан статичный превью Яндекс-карты с меткой студии
  И под картой показан текстовый блок адреса студии
  Когда клиент тапает по карте или «Открыть карту»
  Тогда открывается шторка BS-004 с интерактивной картой и меткой студии
  И при недоступности карты показан текстовый fallback (адрес + ссылка)

Сценарий: AC-002 CTA «Отменить» доступна для активной предстоящей записи
  Дано запись активна и класс ещё не начался
  Когда экран в состоянии Content
  Тогда кнопка «Отменить» активна (enabled)
  И тап по ней открывает шторку BS-003 «Подтверждение отмены»

Сценарий: AC-003 CTA недоступна для прошедшего класса (UC-2 E1)
  Дано время старта класса уже наступило
  Когда экран в состоянии Content
  Тогда кнопка «Отменить» показана неактивной (disabled)
  И рядом показано пояснение, что отменить запись нельзя

Сценарий: AC-004 CTA недоступна для отменённой записи (UC-2 E2)
  Дано запись уже отменена (ранняя, поздняя или студией)
  Когда экран в состоянии Content
  Тогда кнопка «Отменить» показана неактивной (disabled) с пояснением «Запись уже отменена»
  И отображается текущий статус и дата отмены

Сценарий: AC-005 Запуск подтверждения отмены
  Дано кнопка «Отменить» активна
  Когда клиент нажимает «Отменить»
  Тогда открывается шторка BS-003 с правилом 24 часов
  И отмена не выполняется без подтверждения в шторке

Сценарий: AC-006 Статус считывается без цвета
  Дано экран в состоянии Content
  Когда отображается бейдж статуса
  Тогда смысл статуса передан иконкой и/или текстом, а не только цветом
```

## 11. Решения

| # | Решение |
|---|---------|
| 1 | **Отмена — только целиком.** Частичная отмена отдельных мест в MVP не поддерживается (§6.3). |
| 2 | **Точное время отсечки показываем** — «до `<старт − 24 ч>`» (вычисляется из `start_at`) (§8). |
| 3 | **Disabled CTA показывается неактивной с обязательным пояснением** причины (не скрывается) (§6.3, §7). |
| 4 | **Доп. действия для прошедших записей** (например, «связаться со студией») — **вне MVP**. |
| 5 | **Отмена студией** — отображается статус `cancelled_by_studio` с причиной, кнопка отмены недоступна (§6.2, §7). |
