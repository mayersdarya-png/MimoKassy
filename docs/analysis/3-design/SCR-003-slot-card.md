## 1. Назначение и контекст

Экран показывает **все параметры одного класса (слота)**, чтобы клиент мог принять решение
о записи и перейти к оформлению. Это промежуточный шаг между списком классов
([SCR-002](SCR-002-slot-list.md)) и оформлением записи ([SCR-004](SCR-004-booking.md)) —
вложенный экран без таб-бара (см. [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона)).

Контекст использования — запись на ходу, часто с телефона, одной рукой,
в транспорте или между делами (см. [foundations §1](00-foundations.md#1-продукт-и-аудитория)):
клиенту нужно быстро считать ключевые параметры (когда, где, сколько стоит, есть ли места)
и решить, записываться ли.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** убедиться, что класс подходит (время, программа, шеф, цена),
  и понять, есть ли свободные места, чтобы записаться.
- **Задача экрана:** дать полную и честную картину слота за один экран и предоставить
  один очевидный следующий шаг — «Записаться» — либо понятно объяснить, почему запись
  сейчас невозможна.
- **User story:** US-4 — «Как клиент, я хочу видеть все детали класса перед записью,
  чтобы выбрать подходящий слот».

---

## 3. Навигация

**Входящие переходы:**
- Тап по карточке класса в списке [SCR-002](SCR-002-slot-list.md).

**Исходящие переходы:**
- **«Записаться»** (основной CTA) → [SCR-004 Оформление записи](SCR-004-booking.md).
  Активен только при наличии свободных мест (см. §6, §8).
- **Тап по карте студии / «Открыть карту»** → шторка [BS-004 «Карта студии»](BS-004-studio-map.md)
  (открытая интерактивная карта с меткой студии).
- **Назад** (в хедере / системный жест) → возврат к списку [SCR-002](SCR-002-slot-list.md).

Полная карта переходов — в [фича-листе §3](../5-mobile-app-spec/feature-list.md#3-карта-навигации).

## 4. Контент-инвентарь

Все значения приходят из данных слота/программы/шефа (read-only API). Лимиты и числа
**не хардкодятся** — подставляются из данных
(см. [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).

| # | Поле (модель данных) | Что показываем | Обяз. | Примечание |
|---|----------------------|----------------|:-----:|------------|
| 1 | `slot.start_at` | Дата и время старта | Да | Крупно, контрастно (ключевое число). Дата + время. |
| 2 | `program.name` | Название программы | Да | Заголовок параметра «Программа». |
| 3 | `program.type` | Тип программы (простая / сложная) | Да | Бейдж/подпись рядом с названием. |
| 4 | `program.duration_min` | Длительность класса (~3 часа) | Нет | Показывать, **если значение есть** в данных; при отсутствии блок скрывается. |
| 5 | `chef.name` | Имя шефа | Да | Без контактов шефа ([foundations §8.2](00-foundations.md#82-безопасность-данных-в-ui-nfr-11-nfr-12)). |
| 6 | `slot.price` | Цена | Да | Цена за одно место (per-seat). Крупно, контрастно. |
| 7 | `slot.total_seats` / `slot.free_seats` | Всего / свободно мест | Да | Свободно — ключевое число, влияет на состояние CTA. |
| 8 | `slot.free_rental_kits` | Свободно прокатных комплектов | Да | Показывается **числом** свободных прокатных комплектов (из данных, без хардкода); влияет на оформление в SCR-004. |
| 9 | `slot.studio_address` + координаты | Карта студии | Да | **Статичный превью Яндекс-карты с меткой студии** (компонент [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты)). Тап → шторка [BS-004](BS-004-studio-map.md) (интерактивная карта). |
| 10 | `slot.studio_address` | Адрес студии | Да | **Обязательный элемент.** Метка на карте + текстовый блок (адрес студии) под картой. |

> Доступные для записи места рассчитываются ниже по сценарию как
> `min(свободные места, потолок программы, 3)`; на этом экране показываем
> фактические `free_seats` слота, детальный расчёт лимита — на [SCR-004](SCR-004-booking.md).

## 5. Структура и иерархия

Каркас — по [foundations §4.1](00-foundations.md#41-базовый-каркас): фиксированный хедер,
скролл-контент, фиксированный нижний CTA. Таб-бар скрыт (вложенный экран).

- **Хедер (фикс.):** кнопка «Назад» + заголовок экрана (название программы либо «Класс»).
- **Скролл-контент:** блоки параметров слота (§4) сверху вниз по приоритету считывания:
  дата/время → программа+тип (+длительность) → **карта студии + адрес** → шеф →
  места → прокатная экипировка → цена.
- **Нижний CTA (фикс.):** кнопка «Записаться» во всю ширину, всегда видна, не перекрыта
  контентом. Рядом — строка цены/доступности (контекст решения).

## 6. Компоненты и поведение

| Компонент | Поведение |
|-----------|-----------|
| **Кнопка «Назад»** | Возврат на [SCR-002](SCR-002-slot-list.md); тач-зона ≥ минимальной ([foundations §3.1](00-foundations.md#31-тач-зоны-и-размеры)). |
| **Блоки параметров** | Карточки/строки «лейбл + значение». Ключевые числа (дата/время, цена, свободно мест) — крупные и контрастные ([foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)). Каждый блок — самостоятельная единица чтения. |
| **Блок «Тип программы»** | Бейдж с текстовой подписью (не только цветом — [foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)). |
| **Блок «Длительность»** | Отображается только при наличии `program.duration_min`; при отсутствии блок скрывается (без «прочерков»). |
| **Блок «Карта студии»** | Статичный превью Яндекс-карты с меткой студии — компонент [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты). Тап → шторка [BS-004 «Карта студии»](BS-004-studio-map.md) (интерактивная карта). Состояния и fallback — по §4.5. |
| **Блок «Адрес студии»** | **Обязательный.** Текстовый блок (адрес студии, `slot.studio_address`) под картой + метка на карте. Служит текстовым эквивалентом карты (§9). |
| **Блок «Прокатная экипировка»** | Показывает **число свободных прокатных комплектов** из данных, без хардкода (см. §8). |
| **CTA «Записаться» (enabled)** | При `free_seats > 0` — активна, переход на [SCR-004](SCR-004-booking.md). Во всю ширину, высота ≥ тач-зоны. |
| **CTA «Записаться» (disabled)** | При `free_seats = 0` — неактивна, с видимой пояснительной пометкой «Мест нет» (не только цвет/прозрачность — дублируется текстом). Тап не выполняет переход. |

## 7. Состояния экрана

Базовый паттерн — [foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).
Ниже — специфика SCR-003.

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетон в форме будущего контента (блоки параметров + место под CTA). Не пустой экран ([foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана)). |
| **Content** | Все параметры слота (§4) + активный CTA «Записаться». Основной сценарий. |
| **Error** | Заглушка ошибки + кнопка **«Обновить»**. Текст — общий из [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия): «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |
| **Нет свободных мест** (`free_seats = 0`) | Все параметры отображены; CTA **неактивна** с пометкой «Мест нет». Спокойный тон, без давления (P6). |
| **Слот отменён / недоступен** | Параметры показаны с явной пометкой «Класс отменён» / «Слот недоступен»; CTA «Записаться» **неактивна** или скрыта. Объяснение причины нейтральным тоном. |
| **Карта: загрузка / ошибка / офлайн** | По правилам [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты): skeleton при загрузке; при ошибке/офлайн — **fallback на текстовый блок адреса студии** + ссылка «Открыть в Яндекс.Картах». Остальные параметры и CTA «Записаться» остаются доступны. |

## 8. Валидации и микрокопия

Тон и общие тексты — [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).

- **CTA «Записаться» неактивна при `free_seats = 0`.** Пометка рядом/на кнопке: «Мест нет».
  Никаких штрафов и упрёков (P6).
- **Доступность мест:** формулировка вида «Свободно `{free_seats}` из `{total_seats}`» —
  числа подставляются из данных слота, не хардкодятся.
- **Доступность прокатной экипировки:** показывается **числом** свободных прокатных комплектов
  (`slot.free_rental_kits`, из данных, без хардкода); лейблы экипировки — «Своя экипировка» /
  «Прокатная экипировка» ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).
  Точный расчёт лимита `min(свободные комплекты, потолок программы)` применяется на
  [SCR-004](SCR-004-booking.md).
- **Адрес студии (обязательно):** текстовый блок с адресом (`slot.studio_address`)
  под картой + метка на карте. Например: «Адрес студии: ул. Серпуховская, д. 12, лофт 3».
- **Карта студии:** статичный превью Яндекс-карты с меткой студии
  ([foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты));
  тап / «Открыть карту» → шторка [BS-004](BS-004-studio-map.md) с интерактивной картой. Текст адреса
  дублирует карту (доступность, §9).
- **Цена:** показывается как цена **за одно место** (per-seat), значение `slot.price`
  из данных. Итоговая сумма за все места считается на [SCR-004](SCR-004-booking.md).
- **Напоминание об офлайн-оплате:** текст из
  [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия) —
  «Оплата на месте: наличные или перевод на карту.»

## 9. Доступность и mobile-first

Полные правила — [foundations §7](00-foundations.md#7-доступность-nfr-1),
[§3.1](00-foundations.md#31-тач-зоны-и-размеры),
[§3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1).

- Ключевые числа (дата/время старта, цена за место, свободно мест) — крупные и контрастные.
- CTA «Записаться» — крупный, во всю ширину контентной области, тач-зона ≥ минимальной.
- Состояния (тип программы, «Мест нет», «отменено») не передаются только цветом —
  дублируются текстом/иконкой/формой.
- **Карта студии не единственный носитель:** адрес студии продублирован текстом; карта имеет
  доступное имя; при недоступности карты текстовый блок адреса остаётся
  ([foundations §4.5, §7](00-foundations.md#45-карта-адреса-студии-яндекскарты)).
- Все интерактивные элементы имеют доступное имя; layout не ломается при системном
  увеличении шрифта.

## 10. Критерии приёмки дизайна

```gherkin
AC-001: Отображение всех параметров слота
  Дано клиент открыл карточку класса со свободными местами
  Тогда на экране показаны дата и время старта, название и тип программы,
    имя шефа, цена за место, всего и свободно мест и доступность прокатной экипировки
  И длительность программы показана, если значение есть в данных
  И показаны карта студии и адрес

AC-008: Карта студии и адрес
  Дано клиент открыл карточку класса
  Тогда показан статичный превью Яндекс-карты с меткой студии
  И под картой показан текстовый блок адреса студии
  Когда клиент тапает по карте или «Открыть карту»
  Тогда открывается шторка BS-004 с интерактивной картой и меткой студии

AC-009: Карта недоступна — текстовый fallback
  Дано карта не загрузилась (ошибка/офлайн/нет ключа)
  Тогда вместо карты показан текстовый блок адреса студии и ссылка «Открыть в Яндекс.Картах»
  И остальные параметры слота и кнопка «Записаться» остаются доступны

AC-002: Переход к оформлению при наличии мест
  Дано слот имеет free_seats больше нуля
  Тогда кнопка «Записаться» активна
  Когда клиент нажимает «Записаться»
  Тогда происходит переход на экран оформления записи (SCR-004)

AC-003: CTA неактивна при отсутствии мест
  Дано слот имеет free_seats равное нулю
  Тогда кнопка «Записаться» неактивна
  И рядом с кнопкой показана пометка «Мест нет»
  И тап по кнопке не выполняет переход

AC-004: Возврат к списку
  Дано клиент находится на карточке класса
  Когда клиент нажимает «Назад»
  Тогда происходит возврат к списку классов (SCR-002)

AC-005: Состояние ошибки загрузки
  Дано запрос данных слота завершился ошибкой
  Тогда показана заглушка ошибки и кнопка «Обновить»
  Когда клиент нажимает «Обновить»
  Тогда выполняется повторный запрос данных слота

AC-006: Отменённый или недоступный слот
  Дано слот отменён или недоступен для записи
  Тогда показаны параметры слота с пометкой о причине
  И кнопка «Записаться» неактивна или скрыта

AC-007: Напоминание об офлайн-оплате
  Дано клиент видит блок цены
  Тогда показано напоминание «Оплата на месте: наличные или перевод на карту.»
```

## 11. Решения

- **Длительность программы:** показывается **при наличии** `program.duration_min`; при отсутствии блок скрывается (§4, §6).
- **Прокатная экипировка:** показывается **числом** свободных прокатных комплектов (`slot.free_rental_kits`) (§6, §8).
- **Описание программы:** **вне MVP** — отдельного текстового блока с полным описанием блюд нет; на карточке название, тип и краткое меню.
- **Адрес студии:** **обязательный элемент** — метка на карте + текстовый блок (адрес студии, `slot.studio_address`). См. [data-model](../4-design/data-model.md).
- **Карта студии:** статичный превью Яндекс-карты с меткой студии; тап → шторка [BS-004](BS-004-studio-map.md) с интерактивной картой. Компонент — [foundations §4.5](00-foundations.md#45-карта-адреса-студии-яндекскарты). Ключ API Яндекс.Карт — параметр конфигурации.
