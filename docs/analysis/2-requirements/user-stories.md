# Пользовательские истории (User Stories)

## Этап 2. Пользовательские истории. Формат: «Как <роль>, я хочу <действие>, чтобы <ценность>». Каждой истории — критерии приёмки.

**Скоуп:** клиентское мобильное приложение и API для него (роль «Клиент»). Функции шефа и владельца обеспечиваются существующей инфраструктурой и здесь не описываются.

| ID | Как (роль) | Хочу (действие) | Чтобы (ценность) | Критерии приёмки | FR |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **US-1** | Клиент | зарегистрироваться по имени и номеру телефона без пароля | быстро начать пользоваться без барьеров | Указав имя и телефон, получаю доступ; повторный вход — по номеру телефона; сложный пароль не требуется. | FR-1, FR-2 |
| **US-2** | Клиент | видеть список предстоящих слотов классов | выбрать подходящий класс | По умолчанию показываются ВСЕ предстоящие слоты (`only_available=false`), отсортированные по времени старта; слот показывает дату/время, программу (меню), шефа, цену, всего/свободно мест; заполненные (свободно=0) и отменённые слоты остаются в списке с пометкой «Мест нет» и неактивной CTA записи; при отсутствии слотов — пустое состояние. | FR-9 |
| **US-3** | Клиент | фильтровать список слотов | быстро находить подходящий вариант | Могу фильтровать по дате/периоду, типу программы (простая/сложная), наличию свободных мест, шефу; внутри группы (тип программы, шеф — множественный выбор) значения по «ИЛИ», между группами — по «И»; границы дат включительные, в локальной зоне; дефолт — без фильтров; пустой справочник шефов — фильтр скрыт; активные фильтры отображаются; список обновляется по выбору. | FR-38 |
| **US-4** | Клиент | открыть карточку слота со всеми параметрами | понять детали перед записью | Карточка показывает дату/время, программу и её тип (простая/сложная), описание меню, шефа, цену, всего/свободно мест, доступность прокатной экипировки, адрес студии на карте (с fallback на текстовый адрес). | FR-9a, FR-44 |
| **US-5** | Клиент | записаться на выбранный слот | забронировать место на класс | После подтверждения создаётся запись; свободные места уменьшаются; запись появляется в «Моих бронированиях». | FR-10 |
| **US-6** | Клиент | при записи указать свою или прокатную экипировку | прийти со своим инвентарём или взять напрокат | Можно выбрать «своя» / «прокатная» для каждого места; «прокатная» уменьшает прокатный фонд, «своя» — нет. | FR-11, FR-14 |
| **US-7** | Клиент | одной записью забронировать несколько мест (себя + гости) | записать компанию за раз | Можно указать 1–3 места в одной записи при наличии свободных; для каждого гостя выбирается вариант экипировки. | FR-12 |
| **US-8** | Клиент | быть защищённым от записи сверх лимита | не приехать на занятое/несуществующее место | Лимит мест `max_seats = min(free_seats, program.capacity_cap, 3)` (свободные места, потолок программы, лимит 3 на бронь); прокат проверяется отдельно: `rental_count ≤ free_rental_kits`; при гонке запросов двойная бронь/овербукинг исключены. | FR-13, FR-15 |
| **US-9** | Клиент | видеть список своих бронирований | контролировать предстоящие и прошедшие классы | Вижу свои записи (предстоящие и прошедшие) со статусом, параметрами слота, числом мест и вариантом экипировки. | FR-35a |
| **US-10** | Клиент | отменить свою запись до старта | освободить место, если планы изменились | Запись отменяется только целиком (частичная отмена гостя не поддерживается); при ранней отмене (≥ 24 ч, включая ровно 24 ч) освобождаются ВСЕ места и прокатные комплекты брони; при поздней (< 24 ч) запись помечается «поздняя отмена», места не освобождаются, штрафов нет; время в UTC отображается в локальной зоне, статус отмены определяет сервер. | FR-16, FR-17, FR-18 |
| **US-11** | Клиент | видеть цену класса | понимать стоимость заранее | В карточке и при подтверждении показана цена; оплата офлайн (наличные/перевод), запись фиксируется без онлайн-оплаты. | FR-30 |
| **US-12** | Клиент | получать напоминание о предстоящей записи | не забыть и не пропустить класс | Заблаговременно до старта приходит напоминание через системный push (нативное/гибридное приложение, APNs/FCM); SMS/email — Phase 2. (Should) | FR-33 |
| **US-13** | Клиент | просматривать ранее загруженные данные офлайн | ориентироваться при нестабильной связи | При отсутствии сети вижу ранее загруженный список слотов и свои брони с пометкой «данные могли устареть»; изменяющие действия (бронь, отмена) офлайн недоступны с понятным сообщением; единый Error/Retry для повторной загрузки. | FR-9, FR-35a |
| **US-14** | Клиент | быть защищённым от двойной брони при сбое сети | не получить дубль и не переплатить | Клиент при создании брони передаёт обязательный Idempotency-Key; при неопределённом результате повтор с тем же ключом и телом даёт идентичный ответ и не создаёт дубль. | FR-10, FR-13 |
| **US-15** | Клиент | видеть понятные сообщения об ошибках | понять причину и что делать дальше | Для ключевых операций ошибки отображаются по единой матрице (код → реакция): нехватка мест/экипировки, двойная бронь, отменённый/стартовавший слот, неверный код, лимит запросов, серверная ошибка; текст локализован; где уместно — предлагается повтор. | FR-10, FR-16 |
| **US-16** | Клиент | указать аллергии и особенности питания при записи | чтобы шеф и админ знали заранее | В форме записи есть текстовое поле комментария (опционально, до ~500 символов); комментарий сохраняется с бронью и передаётся в админку/шефу; на цену и лимиты не влияет. | FR-50 |
| **US-17** | Клиент | видеть адрес студии на карте в карточке слота | понять, куда идти | В карточке слота отображается адрес студии и координаты на карте (Яндекс.Карты); при недоступности карты — fallback: текстовый адрес без интерактивной карты. | FR-44 |
| **US-18** | Клиент | видеть брони, отменённые студией | понимать, что произошло с классом | Если студия отменила класс (срыв поставки, поломка оборудования), бронь не исчезает, а переходит в статус «Отменена студией» с причиной; приходит push-уведомление; повторная запись на такой слот запрещена (HTTP 410). | FR-46 |
| **US-19** | Клиент | управлять профилем (имя, телефон) и удалять аккаунт | контролировать свои данные | Могу просматривать/редактировать имя и телефон; при удалении аккаунта активные брони аннулируются, прошедшие — анонимизируются; есть возможность logout. | FR-47, FR-48, FR-49 |

## Вне скоупа

Истории шефа и владельца, а также оценки шефов обеспечиваются существующей инфраструктурой и в клиентском приложении не реализуются. Оценки шефов — Phase 2 (согласовано с заказчиком).
